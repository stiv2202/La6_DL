Preparando la Data

El conjunto de datos MNIST consta de imgenes en escala de grises de 28 28 pxeles de dgitos escritos a mano del 0 al 9. Para usarlos con PyTorch, ser necesario realizar algunas conversiones. Para ello, se define transform, una funcin que se utilizar al cargar los datos:

La funcin tiene dos partes:

  • transforms.ToTensor() convierte los datos en un tensor de PyTorch.
  • transforms.Normalize() convierte el rango de los coeficientes del tensor.

Los coeficientes originales proporcionados por transforms.ToTensor() varan de 0 a 1, y dado que los fondos de las imgenes son negros, la mayora de los coeficientes son iguales a 0 cuando se representan utilizando este rango.

transforms.Normalize() cambia el rango de los coeficientes a -1 a 1 restando 0.5 de los coeficientes originales y dividiendo el resultado por 0.5. Con esta transformacin, el nmero de elementos iguales a 0 en las muestras de entrada se reduce drsticamente, lo que ayuda en el entrenamiento de los modelos.

Los argumentos de transforms.Normalize() son dos tuplas, (M, ..., M) y (S, ..., S), donde n representa el nmero de canales de las imgenes. Las imgenes en escala de grises como las del conjunto de datos MNIST tienen solo un canal, por lo que las tuplas tienen solo un valor. Luego, para cada canal i de la imagen, transforms.Normalize() resta M de los coeficientes y divide el resultado por S.

Luego se pueden cargar los datos de entrenamiento utilizando torchvision.datasets.MNIST y realizar las conversiones utilizando transform

El argumento download=True garantiza que la primera vez que se ejecute el cdigo, el conjunto de datos MNIST se descargar y almacenar en el directorio actual, como se indica en el argumento root.

Despus que se ha creado train_set, se puede crear el cargador de datos como se hizo antes en la parte 1.

Cabe decir que se puede utilizar Matplotlib para trazar algunas muestras de los datos de entrenamiento. Para mejorar la visualizacin, se puede usar cmap=gray_r para invertir el mapa de colores y representar los dgitos en negro sobre un fondo blanco:

Como se puede ver ms adelante, hay dgitos con diferentes estilos de escritura. A medida que la GAN aprende la distribucin de los datos, tambin generar dgitos con diferentes estilos de escritura.

Implementando el Discriminador y el Generador

En este caso, el discriminador es una red neuronal MLP (multi-layer perceptron) que recibe una imagen de 28 28 pxeles y proporciona la probabilidad de que la imagen pertenezca a los datos reales de entrenamiento.

Para introducir los coeficientes de la imagen en la red neuronal MLP, se vectorizan para que la red neuronal reciba vectores con 784 coeficientes.

La vectorizacin ocurre cuando se ejecuta .forward(), ya que la llamada a x.view() convierte la forma del tensor de entrada. En este caso, la forma original de la entrada "x" es 32 1 28 28, donde 32 es el tamao del batch que se ha configurado. Despus de la conversin, la forma de "x" se convierte en 32 784, con cada lnea representando los coeficientes de una imagen del conjunto de entrenamiento.

Para ejecutar el modelo de discriminador usando la GPU, hay que instanciarlo y enviarlo a la GPU con .to(). Para usar una GPU cuando haya una disponible, se puede enviar el modelo al objeto de dispositivo creado anteriormente.

Dado que el generador va a generar datos ms complejos, es necesario aumentar las dimensiones de la entrada desde el espacio latente. En este caso, el generador va a recibir una entrada de 100 dimensiones y proporcionar una salida con 784 coeficientes, que se organizarn en un tensor de 28 28 que representa una imagen.

Luego, se utiliza la funcin tangente hiperblica Tanh() como activacin de la capa de salida, ya que los coeficientes de salida deben estar en el intervalo de -1 a 1 (por la normalizacin que se hizo anteriormente). Despus, se instancia el generador y se enva a device para usar la GPU si est disponible.

Para generar dgitos escritos a mano, es necesario tomar algunas muestras aleatorias del espacio latente y alimentarlas al generador.

Se pueden mejorar los resultados considerando ms pocas de entrenamiento. Al igual que en la parte anterior, al utilizar un tensor de muestras de espacio latente fijo y alimentarlo al generador al final de cada poca durante el proceso de entrenamiento, se puede visualizar la evolucin del entrenamiento.

Se puede observar que al comienzo del proceso de entrenamiento, las imgenes generadas son completamente aleatorias. A medida que avanza el entrenamiento, el generador aprende la distribucin de los datos reales y, a algunas pocas, algunos dgitos generados ya se asemejan a los datos reales.
